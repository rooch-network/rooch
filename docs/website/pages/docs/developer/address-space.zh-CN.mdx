# Rooch 地址空间

Move 语言中，模块被定义在地址上，模块依赖地址作为基础标识。Rooch 是一个面向多链的基础设施平台，我们特意规范了地址的分配，既能够避免模块的命名冲突，也方便了扩展。

## Move 标准库地址

Rooch 是一个基于 Move 的 Layer2，因此 Rooch Move 中的标准库沿用 Move 约定的地址 `0x1`，命名地址是 `MoveStdlib`。

## MoveOS 库地址

MoveOS 库基于 Move 标准库提供了更多的扩展，比如状态存储抽象，Table、TypeTable、Object、ObjectStorage、AccountStorage 等数据结构的封装抽象。

为了方便拉取 Move 标准库的上游代码，保持兼容地升级，有必要将 MoveOS 库分配一个独立的地址空间，Rooch 约定 `0x2` 作为 MoveOS 的地址，命名地址是 `MoveosStdlib`。

## RoochFramework 地址

`RoochFramework` 使用地址 `0x3`，包含 Rooch 的账户（Account）、交易验证、多链地址映射等模块。
